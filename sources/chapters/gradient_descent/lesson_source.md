# 梯度下降与优化：单章源稿

## 本章核心问题

学生需要理解：模型并不是“突然学会”的，而是在损失函数给出的误差信号下，沿着能最快降低损失的方向反复修正参数。

## 四个必须回答的问题

1. 学生为什么需要学梯度？
   因为梯度告诉模型参数应该朝哪个方向移动，才能让误差下降得最快。
2. 这个概念解释了 AI 模型里的哪个关键设计？
   它解释了训练循环中的参数更新规则：计算损失、求梯度、按学习率更新参数。
3. 学生如何通过实验看到数学概念在起作用？
   通过可视化二次损失函数上的参数轨迹，观察不同学习率下损失下降、震荡或发散。
4. 老师如何判断学生是否真正理解？
   看学生能否解释梯度方向、学习率过大/过小的后果，并能把实验曲线和更新公式对应起来。

## 数学主线

从一维目标函数开始：

`L(w) = (w - 3)^2 + 1`

导数为：

`dL/dw = 2(w - 3)`

梯度下降更新为：

`w_{t+1} = w_t - eta * dL/dw`

其中 `eta` 是学习率。负梯度方向是局部下降最快的方向，但学习率决定每一步走多远。

推广到多参数模型：

`theta_{t+1} = theta_t - eta * grad_theta L(theta_t)`

这就是大多数模型训练循环中参数更新的抽象骨架。

## 课堂实验

实验使用 NumPy 和 Matplotlib 构造一维二次损失，比较不同学习率下的收敛轨迹。

学生应观察：

- 学习率合适时，损失稳定下降。
- 学习率过小时，下降很慢。
- 学习率过大时，可能震荡甚至发散。

## 课堂提醒

梯度下降不是万能保证。对于非凸损失，可能遇到局部极小、鞍点、平台区和病态曲率。本章只要求学生先理解“误差如何变成参数更新方向”。
